# SharpStream

**P2P-стриминг экрана в 1080p60.**

## Ключевые особенности
* 🎥 Нативный захват экрана в 1080p при 60 кадрах в секунду.
* ⚡ Передача видео и аудио через WebRTC. Сервер нужен только для первоначального "рукопожатия" (Signaling), весь тяжелый трафик идет напрямую между клиентами.
* 🛠 Использование Windows Media Foundation для энкодинга (поддержка NVENC/AMF/QuickSync).
* 🔊 Перехват системного аудио через WASAPI Loopback без необходимости устанавливать виртуальные аудиокабели.
* 🖥️ Интерфейс на базе WPF с архитектурой MVVM.

## Технологический стек
Проект полностью написан на C# и разделен на две логические части:

**Desktop Client (Стример/Зритель):**
* **Фреймворк:** .NET 8 / WPF
* **Архитектура:** MVVM (CommunityToolkit.Mvvm)
* **Захват видео:** Windows.Graphics.Capture API (WGC) + Direct3D 11
* **Захват аудио:** NAudio (WASAPI)
* **Сеть/Транспорт:** WebRTC (SIPSorcery / WebRTC API)

**Signaling Server (Бэкенд):**
* ASP.NET Core Web API
* SignalR (для обмена SDP-офферами и ICE-кандидатами)
